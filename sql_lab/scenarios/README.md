# Scenario Format

Каждый сценарий - это пара файлов:
- `name.before.sql`
- `name.after.sql`

Правила:
- Последний SQL statement считается основным benchmark-запросом.
- Все statements до него считаются setup-командами (например, `CREATE INDEX`).
- Старайся, чтобы before/after решали одну и ту же бизнес-задачу.

Мини-чеклист перед добавлением нового кейса:
- Есть измеримая гипотеза ускорения.
- Есть понятное объяснение оптимизации.
- After-запрос возвращает тот же смысловой результат.
